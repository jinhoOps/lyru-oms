# Lyru OMS Context

Lyru OMS의 주문 운영 도메인에서 사용하는 핵심 용어를 정리한다. 이 문서는 구현 세부사항이 아니라 프로젝트 안에서 같은 말을 같은 의미로 쓰기 위한 용어집이다.

## Language

**주문 표준화 MVP**:
여러 메시지 채널에서 받은 주문/문의 원문을 보존하면서, 고객명, 연락처, 주문 내용, 희망일처럼 운영에 필요한 항목을 가능한 만큼 구조화하는 첫 MVP다. 제작/출고 상태 관리보다 앞선 주문 수집과 정리 흐름이다.
_Avoid_: 주문 대장 MVP, 제작/출고 MVP

**주문/문의 원문**:
카카오톡 채널, 인스타그램 DM, 네이버 톡톡 등에서 사장님이 복사해 온 고객 메시지의 원래 내용이다. 자동 추출이 실패해도 잃지 않고 남아 있어야 하는 기준 기록이며, 기본 목록에 전체 노출하지 않는다.
_Avoid_: 파싱 결과, 주문 데이터

**정보 부족**:
주문을 운영 판단에 쓰기 위해 필요한 항목이 비어 있거나 애매한 상태다. 저장 실패가 아니라 사장님이 추가 확인해야 하는 표시이며, 어떤 항목을 필수로 볼지는 사장님이 관리할 수 있어야 한다.
_Avoid_: 오류, 실패

**저장 우선 원칙**:
필수 정보가 빠지거나 파싱이 실패해도 주문/문의 원문은 저장해야 한다. Phase 1의 우선순위는 완성된 주문만 받는 것이 아니라 흩어진 원문을 잃지 않는 것이다.
_Avoid_: 필수값 누락 저장 차단

**확인 필요**:
정보 부족, 대량 주문, 맞춤 요청처럼 사장님의 판단이나 고객 재확인이 필요한 주문/문의 상태다. 제작/출고 상태가 아니라 수집된 주문/문의의 검토 상태다.
_Avoid_: 제작 준비, 발송 준비

**주의 경고**:
정보 부족과 확인 필요를 같은 경고 레벨로 묶어 보여주는 UI 신호다. 두 사유는 분리해서 기록하되, 사장님이 먼저 확인해야 하는 주문/문의라는 우선순위는 같다.
_Avoid_: 오류, 실패, 차단

**중복 원문 경고**:
같거나 비슷한 주문/문의 원문이 이미 저장되어 있을 때 보여주는 주의 경고다. 실제 중복 주문인지 보완 메시지인지는 사장님이 판단하므로 저장을 차단하지 않는다.
_Avoid_: 중복 저장 차단, 자동 병합

**수동 수정값 우선**:
사장님이 직접 수정한 구조화 항목은 자동 추출값보다 신뢰도가 높은 값으로 본다. 원문 수정이나 재파싱이 있어도 자동으로 덮어쓰지 않고, 차이가 있으면 무채색 점과 터치/호버 설명으로만 알려준다.
_Avoid_: 재파싱으로 수동 입력 덮어쓰기

**재파싱 차이 표시**:
수동 수정값과 새 추출값이 다를 때 필드 옆에 보이는 무채색 참고 표시다. 마우스를 올리거나 모바일에서 터치하면 "원문에서는 이렇게 가져왔어요"와 새 추출값을 보여준다.
_Avoid_: 경고 뱃지, 오류 표시, 강한 색상

**확인필요 상태 전환**:
정보 부족이나 확인필요 사유가 하나라도 있으면 주문/문의 상태를 자동으로 확인필요로 바꾸는 규칙이다. 사유는 분리해서 남기고, 사장님이 확인 후 정리 완료로 바꿀 수 있어야 한다.
_Avoid_: 수집 상태에 경고만 붙이기

**수동 확인필요 전환**:
사장님이 자동 판정 사유가 없어도 주문/문의를 직접 확인필요 상태로 바꾸는 예외적 조작이다. 자주 쓰는 주 흐름은 아니지만, 애매한 고객 응대나 나중에 다시 봐야 하는 주문을 놓치지 않기 위해 허용한다.
_Avoid_: 필수 주 작업, 복잡한 검토 워크플로

**대량 주문 기준**:
상품 DB가 없는 Phase 1에서 대량 주문으로 볼지 판단하는 운영 설정이다. 기본값은 수량 5개 이상으로 두되, 실제 운영 기준은 사장님에게 확인해 조정한다.
_Avoid_: 고정 상품 개수 기준, 개발자가 임의로 확정한 기준

**사장님 확인 질문 화면**:
Phase 1 검증 기간에만 주문 수집 화면 상단에 보여주는 접이식 임시 쪽지형 화면이다. 사장님에게 운영 기준을 쉬운 말로 물어보고, 답변은 앱에 저장하지 않으며 사장님이 개발자에게 직접 연락해 전달한다.
_Avoid_: 정식 운영 화면, 개발자용 설정 화면, 설문 응답 저장 기능

**필수 항목 설정**:
주문/문의에서 어떤 구조화 항목이 비어 있을 때 정보 부족으로 볼지 정하는 운영 설정이다. 기본값은 고객명, 연락처, 주문 내용, 수량, 희망일, 수령 방식이며, 주소는 택배일 때만 필수이고 픽업 시간은 픽업일 때만 필수다.
_Avoid_: 하드코딩된 필수값, 고정 검증 규칙

**라벨 기반 추출**:
주문/문의 원문에서 성함, 연락처, 주문 내용처럼 미리 정한 라벨과 연관 키워드를 찾아 구조화 항목을 채우는 Phase 1 추출 방식이다. 자유 문장을 추정해서 채우기보다 명확한 키워드 근거가 있는 값만 채운다.
_Avoid_: 자유 문장 추정, AI 자동 판정

**수집**:
주문/문의 원문을 붙여넣고 저장한 상태다. 아직 확인이나 정리가 끝났다는 의미는 아니다.
_Avoid_: 신규, 접수됨

**정리 완료**:
사장님이 주문/문의에 필요한 정보가 충분하다고 판단해 정리를 마친 상태다. 제작이나 발송이 끝났다는 뜻은 아니다.
_Avoid_: 완료, 발송 완료, 제작 완료

**정리 완료 유지 규칙**:
정리 완료 상태는 사장님의 수동 확인이 끝났다는 의미이므로, 필수 항목 설정이나 대량 주문 기준이 바뀌거나 주문 필드가 수정되어도 자동으로 확인필요로 되돌리지 않는다.
_Avoid_: 설정 변경 시 과거 주문 자동 재오픈

**연관 키워드 세트**:
라벨 기반 추출에 쓰는 필드별 키워드 목록이다. Phase 1에서는 사장님이 직접 수정하지 않고 개발자가 사장님 피드백을 받아 코드나 문서에서 관리한다.
_Avoid_: 사장님용 키워드 설정, 운영자 파싱 규칙 관리

**고객 요청사항**:
고객이 주문/문의에서 남긴 요청 내용이다. 사장님 내부 메모와 별도 입력 영역으로 분리하며, 향후 고객 확인이나 출고 전 체크에 사용할 수 있는 고객 발화 기준 정보다.
_Avoid_: 내부 메모, 비고

**사장님 내부 메모**:
사장님만 보는 운영 비고다. 고객 요청사항과 별도 입력 영역으로 분리하며, 목록에서는 내용이 아니라 존재 여부 정도만 노출한다.
_Avoid_: 고객 요청사항, 고객에게 보낼 문구
