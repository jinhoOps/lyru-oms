# Requirements: Lyru OMS

**Defined:** 2026-06-29
**Core Value:** 사장님이 오늘 만들고 보내야 할 주문과 변경 요청을 놓치지 않고 확인할 수 있어야 한다.

## v1 Requirements

### 주문 수집과 등록

- [ ] **ORD-01**: 사용자는 주문을 수동으로 등록할 수 있다.
- [ ] **ORD-02**: 사용자는 네이버 스마트스토어 등 외부 주문 목록을 CSV 또는 엑셀에서 가져올 수 있다.
- [ ] **ORD-03**: 사용자는 주문 출처를 네이버 스마트스토어, 네이버 톡톡, 카카오톡 채널, 인스타그램, 기타로 구분할 수 있다.
- [ ] **ORD-04**: 사용자는 고객명, 연락처, 주문 상품, 수량, 희망 발송일, 수령/배송 메모를 주문에 기록할 수 있다.

### 일정과 생산 관리

- [ ] **SCH-01**: 사용자는 오늘, 내일, 이번 주 희망 발송 주문을 볼 수 있다.
- [ ] **SCH-02**: 사용자는 날짜별 주문 수량과 제작 물량을 확인할 수 있다.
- [ ] **SCH-03**: 사용자는 하루 생산 가능량을 넘는 날짜를 위험 상태로 확인할 수 있다.
- [ ] **SCH-04**: 사용자는 발송일 기준으로 주문을 정렬하고 필터링할 수 있다.

### 요청사항과 변경 관리

- [ ] **REQ-01**: 사용자는 주문별 요청사항을 체크리스트로 기록할 수 있다.
- [ ] **REQ-02**: 사용자는 고객 요청 변경을 변경 전/변경 후/요청 채널/확인 여부와 함께 기록할 수 있다.
- [ ] **REQ-03**: 사용자는 확인되지 않은 요청 변경이 있는 주문을 별도로 볼 수 있다.
- [ ] **REQ-04**: 사용자는 발송 완료 전에 미확인 요청사항이 남아 있으면 경고를 볼 수 있다.

### 주문 상태와 작업 흐름

- [ ] **STS-01**: 사용자는 주문 상태를 신규, 확인 필요, 제작 준비, 발송 준비, 발송 완료, 취소로 변경할 수 있다.
- [ ] **STS-02**: 사용자는 상태별 주문 목록을 볼 수 있다.
- [ ] **STS-03**: 사용자는 주문 상세 화면에서 상태 변경 이력을 볼 수 있다.
- [ ] **STS-04**: 사용자는 오늘 처리해야 할 주문만 모아 보는 작업 화면을 사용할 수 있다.

### 모바일 사용성

- [ ] **MOB-01**: 사용자는 모바일 화면에서 주문 목록, 주문 상세, 요청사항 체크를 사용할 수 있다.
- [ ] **MOB-02**: 사용자는 모바일에서 한 손 조작에 적합한 주요 상태 변경 버튼을 사용할 수 있다.
- [ ] **MOB-03**: 사용자는 검색과 필터를 통해 고객명, 연락처, 주문 출처, 발송일로 주문을 찾을 수 있다.

### 데이터 안정성

- [ ] **DATA-01**: 사용자는 주문 데이터를 내보내 백업할 수 있다.
- [ ] **DATA-02**: 사용자는 실수로 변경한 주문을 추적할 수 있도록 주요 변경 이력을 볼 수 있다.
- [ ] **DATA-03**: 시스템은 필수 정보가 빠진 주문을 저장하기 전에 명확히 표시한다.

## v2 Requirements

### 채널 연동

- **INT-01**: 사용자는 네이버 스마트스토어 주문을 자동으로 동기화할 수 있다.
- **INT-02**: 사용자는 카카오톡 채널 문의를 주문 후보로 자동 수집할 수 있다.
- **INT-03**: 사용자는 인스타그램 DM 요청을 주문 변경 후보로 연결할 수 있다.

### 운영 확장

- **OPS-01**: 사용자는 원데이 클래스 예약과 주문 일정을 함께 볼 수 있다.
- **OPS-02**: 사용자는 상품별 제작 소요 시간과 생산 가능량을 설정할 수 있다.
- **OPS-03**: 사용자는 월별 매출, 객단가, 재구매 고객을 확인할 수 있다.

## Out of Scope

| Feature | Reason |
|---------|--------|
| 고객용 쇼핑몰 | v1은 내부 운영 안정화가 목표다 |
| 다중 직원 권한 관리 | 1인 운영자를 위한 단순성이 우선이다 |
| 실시간 채팅 답변 자동화 | 주문 누락 방지보다 복잡하고 브랜드 리스크가 크다 |
| 정산/세무 자동화 | 별도 전문 영역이며 v1 핵심 가치와 거리가 있다 |
