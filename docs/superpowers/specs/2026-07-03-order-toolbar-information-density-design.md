# 주문 목록 도구와 정보 밀도 정리 설계

## 목표

주문 목록의 헤더, 도구 버튼, 상태 표시, 카드/목록 행 정보를 정리해서 모바일에서도 줄바꿈이 예측 가능하고 중복 정보가 줄어든 화면을 만든다.

## 확정 요구사항

- 앱 상단 헤더에는 질문 쪽지와 톱니바퀴 관리 버튼만 둔다.
- 정렬과 보기 전환은 주문 목록 헤더의 도구로 둔다.
- 주문 수는 `주문 목록` 제목 옆에 배치한다.
- 주문 수집 영역은 접고 펼칠 수 있어야 하며, 접힘 상태는 draft 성격으로 `localStorage`에 저장한다.
- 중복 정보 제공 pill은 제거한다. `확인 필요` 같은 총괄 pill을 반복하지 않고, 필요한 경우 `정보 n개`, `확인 n개`처럼 짧은 집계만 보여준다.
- 주문 행은 상단 뱃지, 중앙 주문 내용, 하단 필드 요약 구조로 정리한다.
- `발송 완료`, `확인 필요`, `제작 준비` 등 상태에 따라 `statusPill`과 주문 행 테두리 색을 구분한다. 발송 완료는 낮은 채도로 표시한다.
- `관리 설정` 텍스트 버튼은 톱니바퀴 아이콘 버튼으로 바꾼다.

## 컴포넌트 설계

- `App.tsx`
  - 관리 버튼의 접근 가능한 이름은 유지하되 화면에는 톱니바퀴만 보이게 한다.
  - 주문 수집 패널의 접힘 상태를 `lyru-oms.capturePanel.collapsed.v1` 키에 저장한다.
  - 주문 수집 헤더에는 제목, 설명, 출처 선택, 접기/펼치기 버튼을 배치한다.
- `OrderList.tsx`
  - 주문 목록 제목 옆에 표시 건수를 붙인다.
  - 출처 필터, 정렬 버튼, 보기 버튼을 주문 목록 전용 도구로 유지한다.
  - 정렬 버튼은 작은 텍스트 `정렬`을 사용하고 메뉴의 라디오 선택은 유지한다.
  - 보기 버튼은 작은 텍스트 `보기`를 사용하고 메뉴에서 카드형/목록형을 선택한다.
  - 목록/카드 행 모두 상단 배지, 중앙 주문 내용, 하단 필드 요약 흐름을 따른다.
  - 총괄 `확인 필요` pill은 제거한다.

## 테스트와 검증

- 컴포넌트 테스트로 관리 버튼 이름, 주문 수집 접기/펼치기 저장, 정렬/보기 버튼 동작, 중복 pill 제거, 상태별 class를 확인한다.
- 전체 테스트, 빌드, `git diff --check`를 실행한다.
- 실제 Vite 화면을 데스크톱과 모바일 크기에서 확인한다.
