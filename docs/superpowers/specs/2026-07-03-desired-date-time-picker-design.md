# 희망일/시간 선택기 설계

## 목표

주문 상세에서 희망일과 픽업 시간을 텍스트로 직접 입력하게 두지 않고, 모바일에서도 빠르게 고를 수 있는 바텀시트형 선택 흐름을 제공한다.

## 결정

- 날짜 선택은 `react-day-picker`를 사용한다.
- 앱 UI는 기존 상세 모달 안에서 뜨는 바텀시트로 맞춘다.
- 픽업 주문은 `희망일/시간` 하나의 필드에서 날짜와 픽업 시간을 함께 선택한다.
- 택배 주문은 `희망일` 날짜만 선택하고 시간 선택 영역은 보여주지 않는다.
- 적용 시 `desiredDateTime`, `pickupTime`, `parsedDate`, `manuallyEditedFields`를 같은 사용자 행동의 결과로 갱신한다.

## 비목표

- 운영 시간 설정, 매장 휴무일, 생산 가능 수량 제한은 이번 범위에 넣지 않는다.
- 자연어 입력 파서는 유지하되, 상세 모달의 기본 편집 UI는 선택형으로 바꾼다.
