# 사장님 확인 질문 쪽지 팝오버 설계

## 배경

Phase 1 검증 기간에는 사장님에게 확인해야 할 운영 질문을 주문 수집 화면에서 볼 수 있어야 한다. 현재 질문 영역은 주문 수집 폼 위에 접이식 박스로 배치되어 있어 원문 입력과 저장 흐름을 밀어내며 조작 UI를 방해한다.

이 기능은 임시 검증 보조 장치다. 답변 저장, 질문 관리, 별도 설정 기능으로 키우지 않고, 필요한 질문을 확인할 수 있는 작은 쪽지 진입점만 유지한다.

## 목표

- 주문 수집 폼 위의 질문 박스를 제거해 입력 흐름을 방해하지 않는다.
- `관리 설정` 버튼 옆에 쪽지 아이콘 버튼을 배치한다.
- 쪽지 아이콘을 누르면 기존 질문 목록을 작은 팝오버로 보여준다.
- 질문 확인 기능은 유지하되 답변 저장이나 관리 기능은 추가하지 않는다.

## 비목표

- 질문 항목을 사용자가 편집하는 기능은 만들지 않는다.
- 질문 답변을 앱에 저장하지 않는다.
- 알림, 배지, 읽음 상태, 외부 메시지 연동은 포함하지 않는다.
- 관리 설정 모달과 질문 쪽지를 통합하지 않는다.

## 접근안

### 선택안: 헤더 액션의 쪽지 아이콘 팝오버

`appHeader` 오른쪽에 `headerActions` 영역을 만들고, 쪽지 아이콘 버튼과 `관리 설정` 버튼을 나란히 둔다. 쪽지 버튼을 누르면 헤더 오른쪽 기준의 작은 팝오버가 열리며, 안내 문장과 기존 질문 목록을 보여준다.

이 방식은 기존 질문 내용을 유지하면서 주문 수집 폼의 공간을 회복한다. 기능이 임시라는 점에도 맞고, 설정과 분리되어 사용자가 “운영 질문 확인”과 “판정 기준 설정”을 혼동할 가능성이 낮다.

### 대안: 설정 버튼 주변 텍스트 링크

쪽지 아이콘 대신 `확인 질문` 텍스트 버튼을 둘 수 있다. 의미는 명확하지만 헤더가 더 붐비고 임시 기능이 실제 업무 기능처럼 무거워 보일 수 있다.

### 대안: 모바일 하단 시트

모바일에서만 하단 시트로 띄우면 터치 사용성은 좋다. 다만 현재 질문은 세 개뿐이고 임시 기능이므로 별도 반응형 패턴을 추가할 만큼의 가치가 낮다.

## UI 동작

- 기본 상태에서는 쪽지 아이콘 버튼만 보인다.
- 버튼의 접근성 이름은 `사장님께 확인할 질문`으로 둔다.
- 클릭하면 팝오버가 열리고 다시 클릭하면 닫힌다.
- 팝오버에는 짧은 안내 문장과 기존 질문 세 개를 그대로 표시한다.
- 질문 영역은 주문 수집 패널 내부에서 제거한다.
- 팝오버는 헤더 액션 영역 기준으로 오른쪽 정렬한다.
- 모바일에서는 화면 폭을 넘지 않도록 `max-width`와 좌우 여백을 둔다.

## 컴포넌트 구조

- `QuestionNote`는 계속 별도 컴포넌트로 유지한다.
- `QuestionNote` 내부 상태로 팝오버 열림 여부를 관리한다.
- `App`은 `QuestionNote`를 `appHeader`의 액션 영역으로 이동한다.
- `App`에는 `headerActions` 래퍼를 추가해 쪽지 버튼과 `관리 설정` 버튼을 정렬한다.
- 기존 질문 배열은 그대로 `QuestionNote` 내부에 둔다.

이 구조는 임시 기능을 고립시켜 Phase 1 검증이 끝났을 때 제거하기 쉽다.

## 스타일

- 쪽지 버튼은 기존 `.iconButton` 계열을 재사용하되, 헤더에서 과하게 튀지 않도록 보조 버튼 톤을 쓴다.
- 팝오버는 작은 흰색 패널, 8px radius, 얕은 그림자, 차분한 브라운/네이비 텍스트를 사용한다.
- 팝오버가 열려도 레이아웃을 밀지 않는다.
- 본문 텍스트는 13-14px 범위로 유지해 헤더 부속 정보처럼 보이게 한다.

## 오류와 경계 조건

- 질문 목록은 정적 배열이므로 로딩 실패나 저장 실패 상태는 없다.
- 팝오버가 열려도 주문 입력, 목록 선택, 설정 모달 열기는 차단하지 않는다.
- 설정 모달을 열 때 팝오버가 남아 있어도 큰 문제는 없지만, 구현 단계에서 자연스럽게 닫기 쉬우면 닫는다.

## 테스트

- `QuestionNote` 렌더링 시 기본적으로 질문 본문이 보이지 않는지 확인한다.
- `사장님께 확인할 질문` 버튼 클릭 시 질문 목록이 보이는지 확인한다.
- 다시 클릭하면 질문 목록이 사라지는지 확인한다.
- `App` 테스트 또는 컴포넌트 테스트에서 주문 수집 영역 위에 기존 질문 박스가 항상 노출되지 않는 흐름을 확인한다.

## 승인된 범위

사용자는 GitHub Pages 배포 성공 후, 사장님께 확인할 질문 항목이 현재 조작 UI를 해친다고 지적했고 `관리 설정` 옆 쪽지 아이콘 방식의 팝오버를 선택했다. 이후 “임시로 쓰는거랑 알아서해”라고 승인했으므로 기능 확장은 하지 않고 작은 UX 정리로 진행한다.
