# 달력형 기간 모드 설계

## 배경

주문 목록에는 이미 `달력형 보기`가 있다. 하지만 현재 구현은 주문의 `등록일 ~ 마감일` 기간 안에 있는 날짜마다 마커를 하나씩 렌더링한다. 이 방식은 하나의 주문이 여러 날짜에 걸쳐 이어지는 것이 아니라, 여러 개의 주문이 반복해서 있는 것처럼 보이게 만든다.

다음 개선은 달력이라는 인식은 유지하되, 각 주문이 하나의 연속된 기간으로 읽히게 만드는 것이다.

마감일은 기존과 동일하게 `희망일`을 사용한다. 별도 `deadline` 필드를 추가하는 것은 이번 범위가 아니다.

## 목표

- `달력형 보기` 안에 `일별`, `2주`, `월별` 모드를 추가한다.
- 기본 달력 모드는 `월별`로 둔다.
- `2주`는 오늘이 포함된 일요일~토요일 주와 그 다음 일요일~토요일 주를 표시한다.
- 같은 주문을 날짜마다 독립된 칩처럼 반복 표시하지 않는다.
- 주문 기간을 연결된 막대 또는 명확히 이어진 행으로 표시한다.
- 모바일에서도 읽기 쉽고, 화면 가로 넘침이 없게 한다.

## 비목표

- 생산 가능량, 과부하 경고, 날짜별 위험도는 추가하지 않는다.
- 별도 일정 페이지는 만들지 않는다.
- 이번 범위에서는 이전/다음 기간 이동을 추가하지 않는다.
- 새 마감일 필드는 만들지 않는다.
- 파싱 로직이나 확인 필요 판정 로직은 바꾸지 않는다.

## 달력 모드 선택

사용자가 `보기 > 달력형 보기`를 선택하면 달력 영역 상단에 모드 선택 컨트롤을 표시한다.

- `일별`
- `2주`
- `월별`

기본값은 `월별`이다. 이 선택값은 기존 주문 목록 보기 모드와 별도로 저장한다. 사용자가 다시 `달력형 보기`로 돌아오면 마지막으로 선택한 달력 모드가 복원된다.

## 공통 날짜 규칙

- 시작일은 `createdAt`을 Asia/Seoul 기준 날짜로 변환해 사용한다.
- 종료일은 `desiredDateTime`에서 명시 날짜를 우선 사용하고, 값이 비어 있으면 `parsedDate`를 사용한다.
- 상대 날짜이거나 희망일을 계산할 수 없는 주문은 `날짜 확인 필요` 영역에 표시한다.
- 종료일이 시작일보다 빠른 주문도 `날짜 확인 필요` 영역에 표시한다.
- 주 단위 계산은 일요일을 첫 요일로 사용한다.

## 월별 모드

`월별`은 현재 월을 일요일~토요일 고정 그리드로 표시한다.

월 첫 주와 마지막 주를 완성하기 위해 필요한 앞뒤 날짜도 함께 표시한다. 각 날짜 칸에는 날짜 숫자를 표시한다. 해당 월 범위와 겹치는 주문은 연결된 기간 막대로 표시한다.

- 막대는 주문 기간 중 화면에 보이는 첫 날짜에서 시작한다.
- 막대는 주문 기간 중 화면에 보이는 마지막 날짜에서 끝난다.
- 실제 등록일이나 마감일이 화면 안에 있으면 `등록`, `마감` 라벨을 표시한다.
- 중간 구간은 시각적으로 이어져 보이게 하되, 주문 요약을 매 날짜마다 반복하지 않는다.

주문 기간이 여러 주에 걸치면 주 행 단위로 막대가 나뉘어 보일 수 있다. 이때도 같은 주문이라는 것이 보이도록 같은 라벨/색상 처리를 유지한다.

## 2주 모드

`2주`는 일요일~토요일 행을 두 개 표시한다.

- 첫 번째 행: 오늘이 포함된 주
- 두 번째 행: 그 다음 주

예를 들어 오늘이 2026년 7월 6일 월요일이면 다음 범위를 표시한다.

- 1주차: 2026년 7월 5일 일요일 ~ 2026년 7월 11일 토요일
- 2주차: 2026년 7월 12일 일요일 ~ 2026년 7월 18일 토요일

주문은 이 2주 범위 안에서 잘린 연결 막대로 표시한다. 범위 밖의 주문은 그리드에 표시하지 않는다. 단, 희망일을 계산할 수 없는 주문은 `날짜 확인 필요`에 계속 표시한다.

## 일별 모드

`일별`은 기본적으로 오늘 날짜를 보여준다. 오늘이 주문 기간 안에 포함되는 주문만 표시한다.

각 주문은 한 번만 표시하고 상태 라벨을 붙인다.

- `등록`: 오늘이 등록일인 주문
- `마감`: 오늘이 희망일인 주문
- `진행 중`: 오늘이 등록일과 희망일 사이에 있는 주문

일별 모드에서도 같은 주문을 여러 항목으로 반복하지 않는다.

## 데이터 구조

컴포넌트는 먼저 재사용 가능한 달력 기간 데이터를 만든다.

- 유효 기간 항목: 주문, 시작일, 종료일
- 미해결 항목: 주문과 날짜를 계산할 수 없는 이유
- 표시 구간 항목: 주문, 화면에 보이는 시작일, 화면에 보이는 종료일, 구간 위치

현재의 날짜별 marker 구조는 제거하거나 보조 헬퍼로만 낮춘다. 달력 화면은 독립적인 날짜별 marker가 아니라 주문 기간 구간에서 렌더링해야 한다.

es-toolkit은 의미가 분명해지는 곳에만 사용한다.

- `groupBy`: 기간 구간을 주 행이나 날짜 기준으로 묶을 때 사용한다.
- `sortBy`: 날짜순 정렬에 사용한다.

날짜 계산은 작은 로컬 헬퍼가 더 명확하므로 es-toolkit을 억지로 쓰지 않는다.

## UI 세부사항

- 달력 모드 선택 컨트롤은 달력 콘텐츠 바로 위에 둔다.
- 달력 그리드는 `일 월 화 수 목 금 토` 7개 고정 열을 사용한다.
- 기간 막대는 작은 텍스트와 안정적인 높이를 사용해 레이아웃 흔들림을 줄인다.
- 모바일에서 월별/2주 모드는 주 안의 기간 행을 세로로 쌓아도 되지만, 가로 스크롤은 만들지 않는다.
- 주문 막대는 버튼이며 누르면 주문 상세를 연다.
- `날짜 확인 필요` 영역은 현재 달력 그리드 아래에 유지한다.

## 접근성

- 달력 모드 선택 컨트롤은 native radio input 또는 `aria-pressed`가 있는 버튼을 사용한다. 기존 메뉴 패턴에 맞으면 native radio input을 우선한다.
- 각 주문 기간 막대는 주문 요약과 기간 상태를 포함한 접근 가능한 이름을 가진 버튼이어야 한다.
- 날짜 라벨은 색상에만 의존하지 않고 실제 텍스트로 보여준다.
- 연결된 기간 스타일만으로 의미를 전달하지 않는다. 가능한 경우 시작/끝 라벨을 함께 표시한다.

## 테스트

`OrderList` 중심으로 다음 테스트를 추가한다.

- `달력형 보기`의 기본 달력 모드는 `월별`이다.
- 달력 모드를 `2주`로 바꿀 수 있고, 이 값은 목록/카드/달력 보기 모드와 별도로 저장된다.
- `2주`는 오늘이 포함된 주의 일요일부터 다음 주 토요일까지 표시한다.
- 여러 날짜에 걸친 주문은 날짜마다 전체 칩이 반복되지 않고, 보이는 주 행당 하나의 연결된 기간 표현으로 표시된다.
- `일별`은 해당 날짜에 걸친 주문을 한 번만 표시하고 `진행 중`, `등록`, `마감` 중 맞는 상태를 보여준다.
- 희망일이 없거나 상대 날짜인 주문은 계속 `날짜 확인 필요`에 표시된다.

완료 전에는 focused `OrderList` 테스트, 전체 테스트, production build를 실행한다.
