# Lyru OMS

1인 가게 사장님을 위한 주문 관리 시스템입니다. 네이버 스마트스토어, 네이버 톡톡, 카카오톡 채널, 인스타그램 등으로 흩어진 주문과 문의를 한곳에서 확인하고, 희망 발송일과 변경 요청을 놓치지 않게 돕는 것이 목표입니다.

초기 컨셉은 "만능 직원처럼 옆에서 주문을 정리해주는 운영 도구"입니다. 복잡한 협업 시스템이나 자동화 플랫폼보다, 사장님이 프리미엄 수제 디저트 생산에 집중할 수 있도록 오늘 처리해야 할 주문과 확인할 요청을 명확히 보여주는 데 집중합니다.

## 배경

현재 주문과 문의는 여러 채널에 나뉘어 들어올 수 있습니다.

- 네이버 스마트스토어 주문
- 네이버 톡톡 문의
- 카카오톡 채널 문의
- 인스타그램 등 기타 메시지 채널

특히 희망 발송일, 배송 날짜 변경, 포장/수령 요청 같은 정보는 대화 중에 바뀔 수 있어 수기 기록 누락 위험이 큽니다. Lyru OMS는 이 흐름을 한 화면에서 확인하고 체크할 수 있게 만드는 것부터 시작합니다.

## 대상 사용자

주요 사용자는 한국적 베이스의 프리미엄 곶감 디저트와 화과자를 만드는 1인 가게 사장님입니다.

운영 환경의 전제는 단순합니다.

- 제작자는 한 명이다.
- 품질 유지를 위해 하루 생산량에는 한계가 있다.
- 주문 채널은 여러 개지만, 확인 책임은 사장님 한 명에게 집중된다.
- 도구는 본업을 방해하지 않고 기억 부담을 줄여야 한다.

## v1 방향

v1은 내부 운영용 주문 관리에 집중하되, 첫 MVP는 제작/출고 관리보다 앞단의 주문 표준화에 집중합니다. 카카오톡 채널과 인스타그램 DM처럼 메시지로 들어오는 주문/문의 내용을 한곳에 붙여넣고, 원문을 보존하면서 가능한 항목만 구조화해 사장님이 빠르게 훑어볼 수 있게 만드는 것이 첫 목표입니다.

- 카카오톡, 인스타그램, 네이버 톡톡 주문/문의 텍스트 수집
- 붙여넣은 원문 보존과 주문서 항목 자동 추출
- 정보 부족, 대량 주문, 맞춤 요청 등 확인 필요 표시
- 고객 요청사항과 사장님 내부 메모 분리
- 주문 출처, 고객 정보, 상품, 수량, 희망 발송일 기록

이후 단계에서는 다음 기능으로 확장합니다.

- 주문 수동 등록과 외부 주문 목록 가져오기
- 주문별 요청사항과 변경 요청 체크
- 신규, 확인 필요, 제작 준비, 발송 준비, 발송 완료 상태 관리
- 오늘, 내일, 이번 주 발송 물량 확인
- 생산 가능량 초과 위험 표시
- 모바일에서 빠르게 주문 확인과 상태 변경

자동 채널 연동, 구글폼 기반 고객 주문서, 정산/세무, 고객용 주문 페이지, 상세 매출 분석은 초기 범위에서 제외합니다. 먼저 주문/문의가 한 화면에 모였을 때 사장님이 누락 정보와 확인 필요 주문을 빠르게 알아볼 수 있는지 검증합니다.

## 브랜드와 화면 톤

제품의 시각 방향은 화이트와 골드를 기본으로, 브라운 계열 포인트와 다크 네이비를 보조로 사용하는 절제된 프리미엄 감성입니다. 참고 분위기는 오설록, 페레로 로쉐처럼 채도를 낮춘 고급스러운 톤입니다.

다만 화면은 마케팅 페이지가 아니라 운영 도구입니다. 첫 화면은 브랜드 장식보다 새로 들어온 주문/문의, 정보가 부족한 주문, 확인이 필요한 대량/맞춤 요청을 먼저 보여줘야 합니다.

## 운영 원칙

- 사장님의 본업은 생산이며, 시스템은 생산을 방해하지 않아야 한다.
- 자동화보다 누락 방지와 검증 가능한 기록을 먼저 만든다.
- 모바일 웹에서 실제 작업 중 확인할 수 있어야 한다.
- 기능은 단계적으로 고도화한다.
- 매출 다각화나 원데이 클래스 관리는 주문 관리가 안정화된 뒤 확장한다.

## 현재 계획

초기 로드맵은 주문 표준화 MVP를 먼저 검증한 뒤, 제작/출고 관리로 확장합니다.

1. 주문 표준화 MVP
2. 주문 대장과 요청 변경 관리
3. 발송일과 생산량 보기
4. 출고 전 검수와 이력
5. 모바일 운영 최적화

첫 구현 목표는 주문 표준화 MVP입니다. 사장님이 카카오톡, 인스타그램, 네이버 톡톡 등에서 받은 주문/문의 텍스트를 붙여넣으면 원문은 그대로 남기고, 성함, 연락처, 주문 내용, 수량, 희망일, 수령 방식, 요청사항 같은 항목을 가능한 만큼 구조화해야 합니다. 필수 정보가 빠진 주문과 대량/맞춤 요청은 한눈에 확인할 수 있어야 합니다.

## 로컬 실행

```bash
npm install
npm run dev
```

브라우저에서 표시된 로컬 주소를 열면 Phase 1 주문 표준화 MVP를 확인할 수 있습니다.

## Phase 1 확인 흐름

1. 주문 수집 화면 상단의 접이식 쪽지를 확인합니다.
2. 카카오톡, 인스타그램, 네이버 톡톡 원문을 붙여넣고 저장합니다.
3. 필수 정보가 빠져도 주문이 저장되는지 확인합니다.
4. `정보 부족` 또는 `확인필요` 사유가 있으면 상태가 `확인필요`가 되는지 확인합니다.
5. 정보 부족 주문에서 원문을 펼쳐볼 수 있는지 확인합니다.
6. 고객 요청사항과 사장님 내부 메모가 분리되어 있는지 확인합니다.
7. 관리 설정 변경은 이후 수집하거나 수정 저장하는 주문의 기준으로 쓰이며, 기존 주문의 `정리 완료` 상태를 자동으로 되돌리지 않는지 확인합니다.
